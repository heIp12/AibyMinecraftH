# 위협도·분노도 시스템 설계 (괴담=규칙에 얽힌 존재의 육화)

> 상태: 착수(1단계 실구현). 발단 = "기본원칙(괴담은 규칙에 얽혀있다)이 빠져서 생긴 문제"
> — v2 측정에도 남은 운생존(평균 62.5%)·버티기·GM 과보호의 공통 뿌리는 **판 안에서 누적·집행되는
> 괴담 세력 상태가 없다**는 것. 지금은 CorruptionManager(회차 간 메타)·disposition·흩어진 프롬프트
> 의무만 있고, **intra-session 래칫**이 통째로 비어 있었다.

## 0. 핵심 원칙 (1급 불변식)
괴담은 **규칙에 얽힌 존재**다. 두 규칙집합을 구분한다:
- **(A) 괴담 자신의 속박** = 약점/collapse_condition. 괴담을 제 규칙에 걸려 넘어지게 하는 게 파훼.
- **(B) 사람에게 걸린 금기** = trap/taboo. 어기면 괴담이 **강해진다**.
불변식: *남이 규칙을 어기면 괴담은 강해지고, 괴담이 제 규칙을 어기면(또는 어기게 만들면) 붕괴한다.*

## 1. 두 게이지 — 다른 시간축 (확정)
| | **위협도(threat, 세력)** | **분노도(anger, 격앙)** |
|---|---|---|
| 성격 | 느림·거의 영구(래칫) | 빠름·휘발(턴당 감쇠) |
| ↑ 트리거 | 구조적 먹이: 금기위반·**함정**·전파·계약체결·사망·밤 깊어짐·시간끌기 | 직접 도발: 공격·조롱·성역침범·이번 턴 소음 |
| ↓ | 파훼 단계 진척으로만 배수(drain) | 조용히 몇 턴 → 자연 감쇠 |
| 지배 | 피해등급·DC바닥·힌트기근·**클리어 잠금**·강제 격상 | **표적 선정**·이번 비트 즉각성·수동형 발현·**규칙파괴 살해** |
| 스케일 | 0~100 (밴드 40/70/90) | 0~100 (임계 ≈90) |

- 저장: GameStateManager 필드(세이브 포함, turnMode/clock처럼). 스테이지 이동 시 부분 리셋,
  세션 완전 종료 시 초기화.
- **플레이어에겐 숫자 비노출**(메타 노출 금지) — 서술로만 체감. 숫자는 GM 전용 + 뷰어 감사용.

## 2. 사용자 확정 두 메커니즘
### ① 조기종료 함정 → 위협도 ≈90 급등 → 도주 강제
함정(특히 교환·계약형 성급 수락)을 밟으면 위협도가 빠르게 90까지. 정석 클리어 잠금 →
GM은 탈출·생존 프레임으로 선회. **90(=100 아님)** 이라 필사적 고비용엔 부분(B)·생존의 실낱만 남김.
전파형 일회성 탈출게이트를 **보편 임계**로 승격.

### ② 분노도 高 → 규칙 무시 표적 살해 + ★붕괴 결합(필수)★
분노도 ≈90+에서 괴담이 **제 규칙을 어기고** 표적(어그로 축적자)을 죽일 수 있다.
- ★결합★: 괴담이 제 속박을 깨는 순간 = 원칙상 **스스로 취약/붕괴창 개방** + 분노 방출(하락).
  → 단순 즉사 버튼이 아니라 **양날의 절정**: 규칙 준수만으론 안전을 못 사되(진짜 공포), 분노를
  극한으로 몰아 **일부러 규칙을 깨게 만들고 붕괴창에서 파훼**하는 하이리스크 반격이 성립.
- 가드: 근최대 분노에서만·지속 도발로만 도달(사고 불가), 표적=도발한 자, **예고(telegraph)**,
  살해 후 분노 방출로 스팸 불가(절정 1회성).

## 3. 하이브리드 구현 (코드+프롬프트, 기존 패턴 재사용)
clockMinutes·puppetRecoveryTurns·pendingLuckModifier와 동일 패턴:
1. 엔진이 threat/anger 지속 추적(세이브).
2. **엔진 자동 증분**: 이미 감지하는 이벤트 훅(무기 noise, comms_dangerous, 사망, 무행동, 밤 틱).
3. **GM 태그 증분**: `<THREAT +N 이유>` / `<ANGER +N 표적=이름>` → 엔진 커밋·클램프·로깅(뷰어 감사).
4. **매 프롬프트 주입**: "현재 위협도 72/100(격상: 국소 파훼 불가) · 분노도 88/100 표적=백선우".
5. **임계 기계집행(GM 재량 제거)**: 위협≥90 → CLEAR 거부·탈출선회 / 분노≥90 → 규칙파괴 살해+붕괴창 /
   고위협 → 힌트예산 축소·피해 바닥 상승.

## 4. 기존 시스템과 합성(중복계상 금지)
- `scale` = 위협도의 **천장·상승률**(코즈믹일수록 높은 천장, 빠른 상승).
- timeline 사건 = 위협도 **범프**(각본 격상은 게이지 위에 얹힘).
- `CorruptionData`(회차 간 entityMemory) = 위협도 **시작 바닥**("괴담이 널 기억해 더 화나서 시작").
- 정적 disposition = 게이지 반응의 **형태**(분노형=폭발적 / 심판자형=위반 응징 / 굶주린=맹공).

## 5. 단계별 착수 (저위험→고위험)
- **1단계(토대, 저위험)**: 엔진 상태 필드+getter/adjust·세이브/로드·리셋 / GM 프롬프트 밴드 주입 /
  `<THREAT>`·`<ANGER>` 태그 파싱 / 언제 올리는지 프롬프트 규칙 / 무행동·밤 틱 자동증분. #224
- **2단계(임계 집행)**: 위협≥90 CLEAR잠금·탈출선회(전파게이트 보편화) / 분노≥90 규칙파괴 살해+붕괴결합 /
  힌트·피해 스케일. #225
- **3단계(생성 스키마)**: entity에 feed/provoke_triggers·threshold_behaviors·ceiling·drain·rage_break
  슬롯 + 생성기 자기검증(B계열). #226
- **뷰어**: 위협/분노 밴드 타임라인 표시(감사·연출). (3단계 이후)

## 6. 이게 고치는 것
- 함정 무력 → 위반=영구 +위협도 + 임계 잠금(눌어붙음).
- 운생존/버티기 → 정체가 위협도↑ → 못 버팀.
- 양극화 → 연속값 → 중간개입=중간위협=B.
- GM 과보호(#223) → 임계 기계집행으로 소프트 재량 불가.
- 전파 "클리어 불가→탈출"(기존 요구) → 위협 천장=보편 메커니즘으로 승격.
