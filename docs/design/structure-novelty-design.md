# 구조 신박성(Structural Novelty) 시스템 — 설계안 (GPT 조율 대기)

## 배경 / 목표
웹소설 「괴담동아리」·「괴담호텔탈출기」처럼 **매 스테이지(=매 .gdam 생성)가 구조적으로 새롭고,
규모와 함께 사건의 복합도도 계단식으로 커지는** 캠페인을 만든다. 괴물만 바뀌고 공략은 늘
"단서 찾기 → 약점 의식 실행 → 클리어"로 수렴하면 재미가 죽는다.

## 현재 상태(코드 확인)
- 반복 방지가 걸린 축: **괴담 정체·이름**(`recentFamiliar`+`variantBase`, 태그별 7~20창),
  **배역 이름**(`recentNameCsv`), **NPC 개성 어미**(회전 풀).
- 반복 방지가 **없는** 축: **공략의 결**(봉인/격퇴/역이용/탈출/버티기…), **사건 기믹**
  (밀실탈출/시간루프/정체색출/통신유인…). 아키타입 풀은 풍부하나 **매 생성 무기억 샘플링**이라
  한 캠페인(1~5스테이지) 안에서 같은 구조가 괴물만 바꿔 반복될 수 있다.
- 규모 계단 상승은 이미 있음(로컬→코즈믹 단조, DC 하한, "전보다 세졌다" 규칙). **빠진 건 구조 신박성.**
- 방금 확정된 인접 규칙: **해결의 격 = 조건 불리기가 아니다.** 실행은 단순·명료한 행위 1개
  (+긴장 조건 최대 1개)로 유지, 격은 ①진상 파악 난이도 ②실행의 긴장 ③행위의 층위·대가로만.
  진상을 다 알면 그 자리에서 실행·한 장면 결판(조기종결 보존). 수집 체크리스트·다단계 AND 사슬 금지.

## 제안 — 두 축 반복 방지 (기존 이름 no-repeat와 병렬 레인)

### 축1: `clear_shape` (공략의 결 — "어떻게 이기는가", 항상 1개)
enum 후보: 봉인 / 격퇴 / 역이용 / 파훼의식 / 탈출 / 생존·버티기 / 희생 / 자기정화 / 정체폭로 / 진실규명 (≈10종)

### 축2: `structure_motif` (사건 기믹 — "어떤 판으로 굴러가는가", 스테이지 비례 개수)
enum 후보(대부분 이미 아키타입에 존재): 밀실탈출 / 규칙목록준수 / 시간루프 / 정체색출(누가 괴담) /
통신유인 / 인과역전 / 집단환각 / 금지어 / 교환대가 / 정보격리 / 환경침식 / 추격·포위 / 감염확산 /
도덕딜레마 / 역할극강제 … (≈15+종)

### 메커니즘
1. 생성 성공 시 core에 `clear_shape`·`structure_motif[]` 확정(모델 스키마 출력 우선, 없으면 엔진이
   world_rules/collapse_condition/entity.type 키워드로 추론 폴백).
2. 상위 폴더 `.structure_history`에 태그별 최근 창 기록(기존 추적 파일들과 같은 위치).
3. 다음 생성 concept/structure 청크에 주입:
   "이 캠페인에서 이미 쓴 결·기믹은 X·Y다 — **이번엔 둘 다 겹치지 않게** 결이 다른 구조로 짜라."
   (**negative 제약만** — "이걸 써라"가 아니라 "이건 피해라". 개연성을 억지로 눌러 '기믹 위한 기믹'이
   되는 것 방지.)
4. forced/다음 괴담 지정 모드는 무시(기존 패턴).

## 제안 — 복합도 이중 상승 (규모 + 얽힘)
- `structure_motif` **개수**를 스테이지 비례: 1스테이지=1개(순수 단일 기믹) / 2~3=1~2개 /
  4~5=2~3개가 상호작용.
- `clear_shape`는 **항상 1개 유지**(공략은 명료 — 조기종결 원칙). 복합도는 "판의 얽힘"이지
  "클리어 조건 불리기"가 아니다(해결의 격 규칙과 정합). 5스테이지=여러 기믹이 얽히되 진상을 알면
  여전히 한 행위로 결판.

## GPT에 물을 것 (열린 질문)
1. `clear_shape` 확정: 모델 자기신고 vs 엔진 키워드 추론 vs 둘 다(신고 우선+추론 검증)?
   자기신고는 부정확, 키워드 추론은 취약. 어느 조합이 나은가?
2. 반복 방지 창 크기: `clear_shape`는 ≈10종뿐 → 창을 3~4로 좁게(안 그러면 소진). `structure_motif`는
   15+종 → 6~8. 창이 풀보다 커져 소진되면 어떻게 완화(전부 소진 시 가장 오래된 것부터 재허용)?
3. **캠페인 단위 리셋 vs 서버 전역 누적**? 웹소설 느낌의 핵심은 "한 캠페인 안에서 안 겹침"이다 →
   캠페인(새 게임) 시작 시 구조 창을 리셋? 아니면 전역 누적로 서버 전체가 다양?
4. 복합도 상승이 **저사양 GM(haiku급)**에겐 과부하 아닌가(여러 기믹 동시 = 프롬프트·판정 복잡).
   복합도 상한을 모델 티어로 캡?
5. `clear_shape` 다양성 강제가 "생존·버티기"·"희생" 같은 특수 결을 개연성 없이 억지로 끼워
   시나리오를 망치지 않게 하는 안전장치는? (negative 제약만으로 충분한가, 아니면 "안 맞으면 겹쳐도
   됨" 탈출구가 필요한가?)

## 리스크
- **인위성**: 태그 강제가 개연성을 눌러 "기믹 위한 기믹" → negative 제약만, 그리고 "정말 안 맞으면
  개연성 우선" 탈출구.
- **소진**: 5스테이지 캠페인에 결 10·기믹 15면 충분하나 전역 누적이면 금방 소진 → 캠페인 리셋이 유력.
- **격 규칙과의 경계**: 복합도(판의 얽힘) ≠ 클리어 조건 불리기. 프롬프트가 이 경계를 흐리면 억지 조건
  회귀 → "복합도는 사건 전개의 층위, 공략은 여전히 단일 행위"를 프롬프트에 못박기.

## 구현 스케치(승인 후)
- GdamGenerator: 스키마에 `clear_shape`·`structure_motif[]` 필드 + `.structure_history` 기록/조회
  (`recentFamiliar` 미러링) + 생성 청크 주입 + 복합도 개수 스테이지 함수.
- 캠페인 리셋 훅(새 게임 시작 시 구조 창 clear) — TRPGGameManager 게임 리셋 경로.
- lint: `clear_shape` enum 유효성 + 스테이지별 motif 개수 경고(비차단).
