# 게임 템포·정보 경제 재설계 (2026-07)

4패널 다중 에이전트 회의(재미나이 빠른종결 4제안 + GPT Codex 엔진 계약 분석 + 자체 로그 감사)와
사용자 4원칙을 종합해 GM 프롬프트·런타임 주입을 일괄 개편했다.

## 사용자 4원칙 (설계 기준)
1. **단호한 종결** — 끝낼 상황이면 미루지 않고 끝낸다.
2. **힌트 과잉 금지** — 해석·추리는 플레이어 몫. GM이 "저 명부는 죽은 아이들 이름이 틀림없어"처럼
   결론을 서술하면 추리가 사라진다.
3. **행동 결과 보장** — 잘못된 행동은 괴담 자극 + ★사회적 결과★(신고·공권력·평판),
   단서성 행동은 반드시 수확.
4. **알면 쉽고 모르면 맞는 다회차 게임** — 한두 명 사망·전멸은 평범한 결말.

## 핵심: 영감(SPR) 경제 교체 — "해상도" → "연상"

### 왜 바꾸나
구판은 `발견은 보장, 선명도(해상도)는 영감 비례` 사다리(1~2 겉모습 … 15~17 실제 내용 … 18+ 정체)였다.
실로그 감사 결과 두 방향 모두에서 원래 목적(플레이어 추리 보존)을 배반했다:
- **낮은 영감**: 발견 자체가 흐려져("뭔가 긁힌 자국이 있다") 저품질 모델이 같은 저해상도 긁힘을
  여러 턴 재탕하는 **스톤월링**을 낳았다. 영감 15+가 아니면 '실제 내용'을 못 받는 구조라
  평범한 파티는 사실을 영영 못 읽었다.
- **높은 영감**: 아는 시점부터 서술이 과잉으로 붙어(내용+정체까지) **플레이어의 추리를 GM이 대체**했다.
- 영감 조절의 원래 의도는 "아는 시점의 과다 서술 제한"이지 "힌트 발견 제한"이 아니었다.
  힌트 발견 자체는 자주 되어도 좋다.

### 새 경제 (3축 분리)
| 축 | 결정자 | 규칙 |
|---|---|---|
| 발견 성패 | 탐색 행동 | 단서를 찾는 행동이면 발견 보장(스톤월링 금지) |
| 사실 해상도 | (고정) | ★영감 무관★ 관찰 사실(이름·글자·날짜·모양)은 그대로 또렷이 — "김하율이라는 이름이 적힌 이름표가 떨어져 있다". 영감 1~4의 치명적 둔감만 겉모습에 그침 |
| 연상(연결) | 영감(SPR) | 9↓ 없음 · 10~14 관련 단서를 여럿 모았을 때 ★때때로★(모은 단서 수 비례) · 15+ 자주·구체적으로. ★이미 가진 단서끼리만★ 잇고, 스치는 물음 형태로만 |

- **전 구간 단정 금지**: "~틀림없다", "~라는 뜻이다" 류 확정·해석 문장을 지문에 쓰지 않는다.
  사실은 선명하게, 의미는 비워 둔다 — 잇는 것은 플레이어다.
- 생성기 대응: 단서의 **중의성은 내용 자체에 설계**한다(런타임이 흐려 주지 않으므로).

## 적용 변경 (14건 + 정합 3건)

### PromptBuilder.java
1. **루프물 선언**(정체성부): 다회차 전제 — 개별 사망은 평범, 전멸도 결말. 공정성 기준 = **'스탯이나 앎으로
   피할 수 있었던 죽음인가'**. 대부분의 개별 사망은 징조·경고·본인 선택·실패한 판정의 결과.
   **단 아주 드물게 '모르면 당하는' 은닉 치명 함정 허용**(예: 방에 들어서는 순간 매복·저격 즉사) — 공정한 이유:
   ①영감(SPR) 높으면 위험 직감 레이더가 사전 경고(낮으면 '함정에 그대로') ②행운(LUK) 높으면 회피·생존 판정
   통과 ③당해도 다음 회차엔 그 자리를 알고 회피. 그러니 **매복도 영감/행운 판정으로 회피·생존 기회를 한 번은
   줘라**(무판정 통짜 즉사는 관련 스탯 바닥/경고 무시 시만). 남용 금지 — 희소·개연 필수, **전원 몰살만은
   여전히 사전 암시**(무경고 함정으로 파티 전체 지우기 금지).
   - 이 카브아웃은 luckSaves 제거와 정합: LUK 생존은 이제 **투명한 영감/행운 판정**([판정 예비값])으로 흐른다
     (숨은 자동 세이브가 아니라 굴림값이 dc를 넘으면 생존). SPR 회피는 기존 레이더(PB:229 '1~2 함정에 그대로')가
     이미 지원.
2. **정보 경제 교체**(힌트 절제): 발견 보장 + 사실 그대로 + 영감=연상.
3. **단정 화법 금지 불릿** 신설(추리 강탈 금지) — 연상도 "어디서 봤더라 —" 물음형만.
4. **소득 체감에 최소 수확 보장**: 재조사도 헛턴 금지 — 남은 관찰 1개 or "더 없다" 확언.
5. 핵심 단서 규칙 꼬리 정합(영감은 연상이지 자동 진행 아님).
6. **능력치 상세 영감 정의 교체**: 레이더(위험 직감)는 유지, 해상도 사다리 → 연상 게이트.
7. **유령 행운 위기구제 제거**: 프롬프트가 약속하던 "시스템이 4×LUK%로 자동 구제"는
   실코드에 호출부가 없는 dead code(luckSaves)였다 — 약속 삭제 + 메서드 제거.
   생사는 [판정 예비값]이 정하고, 행운은 굴림 보정으로만 작동.
8. **사회적 결과 승격**: 괴담과 별개의 '사회' 축 — 즉시(신고·도망) + 누적(평판→경계→공권력).
   ★개연성 게이트★: 목격→전달 경로가 실재하고 그 무대에 닿는 관할일 때만
   (아무도 없는 곳에 경찰 등장 금지, SCP류 시설엔 일반 경찰이 아니라 제압요원 —
   고립·차단 무대에선 내부 인원의 경계로만).
9. **턴 절차에 '매듭 점검' 단계 신설**(2번): 종결 성립을 매 턴 먼저 확인, ★지난 턴에 놓친
   충족도 소급 <CLEAR>★, 종결 눈앞이면 1~3응답 내 마무리(새 갈래 벌리기 금지).
10. **주사위 절차 통일**: 구 절차 헤더가 "모든 판정은 <DICE>만 내고 다음 응답"(2응답)이라
    상세부(#254 인라인 원칙·굴리는 결정타만 2단계)와 정면 충돌 → 인라인 기본 +
    굴리는 결정타만 2단계로 단일화.
11. 턴 절차 '정보' 단계 문구를 새 경제로 갱신.
- (정합) 관문형 '힌트관문' 문구에서 구 해상도 참조 제거.

### TRPGGameManager.java
12. **collapse_condition 소급 매듭**: 이미 충족됐는데 지나갔으면 늦게라도 <CLEAR>(무효화 금지).
13. **런타임 SPR 사다리 교체**: actorStatGmContext의 7티어 해상도 사다리 → 4티어 연상 게이트

    형용 남발 금지, 수상함은 행동·모순에서 플레이어가 발견.
- (정리) luckSaves dead code 제거.

### GdamGenerator.java

## 회의에서 기각된 제안 (근거 보존)
- **재미나이 ③ NO_HOPE 하향(빠른 전멸)**: 기각 — end_time 경과는 코드상 '생존 D등급' 처리로
  이미 완결이고, 기존 2연속+회복 게이트가 억지 전멸을 막는 설계 그대로 유효.
- **재미나이 ④ Token Waste 메타 키워드**: 기각 — 시스템 키워드의 서술 누출 전례(#213)가 있어
  메타 신호를 프롬프트에 추가하지 않는다. 같은 목적은 '매듭 점검' 절차(9)로 달성.
- **재미나이 ① 루프물 선언**: 순화 채택(1) — 원문은 "죽음 방치"에 가까웠으나 공정성 짝조항을
  붙여 채택.
- **재미나이 ② 즉시 CLEAR**: 이미 6곳에 존재 확인 — 진짜 구멍은 '놓친 종결의 소급'과
  '턴 절차상 점검 부재'였고 (9)(12)로 메움.
