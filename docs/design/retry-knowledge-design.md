# 재도전 지식·자동성공 처리 — 설계안 (GPT 조율 대기)

## 배경
「괴담동아리·괴담호텔탈출기」 감각: **정답을 알아내는 것이 핵심인 괴담에서 '정확한 해법 실행'
자체가 보상**이어야 한다. 재도전은 정공법(적극적 리트라이 = 정보 축적)인데, 지금 엔진은 재도전에서
정보를 재잠금하고, 방금 추가된 무판정 CLEAR 게이트가 '아는 해법의 빠른 실행'까지 주사위로 막는다.

## 코드 검증된 현황
1. **재도전 정보 재잠금(실재)**: `GameStateManager.onRetry()`가 discoveredClues·discoveredFacts를
   전부 clear. 재주입되는 건 NPC 대화 기억뿐(`snapshotNpcMemories`, 오염도 비례). GM 컨텍스트는
   '파티가 정답 단서를 찾았다'를 모른 채 새로 시작 → 정답 기억 플레이어의 정확한 실행이 '미발견'으로
   거절될 위험. (capstone `discoveredAuthoredClueIds`는 재도전 경로에서 미clear라 봉인 해제만 우연히
   유지 — 일관성 없음.)
2. **플레이어 지식 vs 캐릭터 지식(모델 미정의)**: `everKnownContacts`(번호)는 이월되지만
   '발견 단서·사실'은 이월 경로가 없다. 대기실 자유 공유 → 다음 시도 파티 공통 지식이 되는 층이 없음.
3. **정확한 추리에 고난도 주사위(라이브 회귀 — 이번 세션 게이트가 유발)**: `retryThreatFloor()`가
   재도전 시작 위협도를 min(40, level×8)로 올림 → 오염 5단계 재도전은 1턴부터 위협 40. 무판정 CLEAR
   게이트(위협≥40 && DICE 없음 && 최근 3턴 성공판정 없음 → 보류+decisive DICE 요구)가 첫 턴부터
   걸려, 정답을 알고 실행해도 DC 15를 강요.

## 원칙 (사용자 확정 방향)
- **재도전 플레이어 기억 = 정식 자원.** 정답을 기억하면 그 실행을 GM이 '미발견'으로 막지 않는다.
  캐릭터가 어떻게 아는지는 기억·기시감·오염도로 세계관 내 처리(기존 설정 재사용).
- **두 층 분리**: ①플레이어-앎(메타) = '실행·시도할 자격' ②캐릭터-앎(극중) = 서술 연출.
  대기실 공유 정보는 다음 시도에 파티 공통(전원 실행 자격), 극중 인지는 기시감으로 프레이밍.
- **자동성공 우선**: 정확한 해법을 갖춘 실행은 위협도와 무관하게 빠른 결판이 `<DICE>` 의무보다 우선.
  주사위는 ①해법을 모른 채 버티기 ②물리적으로 어려운 실행 보완에만.

## 제안 A — 무판정 CLEAR 게이트 예외 (3번 회귀 해소)
게이트는 '근거 없는 서술만의 클리어'만 잡아야 한다. 아래 중 하나면 게이트 미발동:
- (a) 정답 단서 발견됨: capstone(또는 solution 정의 단서)이 unsealed = 파티가 해법을 손에 넣음.
- (b) 재도전 메타지식: attempts>0이고 파티 기억 원장에 해당 해법 관련 사실이 있음.
- (c) GM 명시 자동성공: CLEAR에 근거 필드(예: `basis:"solution_known"`)를 달아 GM이 '조건 완비 자동
  성공'을 선언 — 감사 가능(악용 시 로그로 드러남).
프롬프트에도 자동성공 우선 규칙을 명문화(위 원칙).

## 제안 B — 파티 기억 원장 (1·2번 해소)
- 발견 단서·핵심사실을 재도전에 이월하는 **persistent 원장**(state에 `retainedFacts` 등, everKnown류).
  discoveredClues의 '서피싱'은 재발견 연출용으로 계속 비워도, 원장은 유지해 GM이 실행을 수용.
- GM 프롬프트에 '파티가 이미 기억하는 사실(재도전 이월)' 섹션 주입 → GM은 이 사실 기반 실행을 막지
  않고, 극중 인지는 기시감으로 서술.
- 대기실 공유: 한 플레이어가 발견→대기 전원 공유한 사실은 다음 시도에 전원 원장에 이월.

## GPT에 물을 것
1. 게이트 예외를 (a)(b)(c) 중 무엇으로? capstone 없는 시나리오는 (a)가 안 걸리는데 폴백은?
2. 재도전에서 '모든' 발견을 이월 vs '해법 관련 핵심'만? 전부 이월하면 재발견 루프·오염도 기억
   메커니즘의 의미가 옅어진다 — 어디까지가 적정?
3. 캐릭터-앎 프레이밍(기시감)을 강제 서술로 넣을지, GM 재량에 둘지?
4. '자동성공 우선'을 너무 열면 저사양 GM이 근거 없이 자동성공을 남발(주사위 0회 회귀) — 이걸 (c)
   근거 필드 의무로 방어하면 충분한가?
5. 파티 공유 층: 대기실 공유를 자동(대기 중 발화 전부 공유)으로 볼지, 명시 공유 행위만 볼지?

## 리스크
- 자동성공 과개방 → 주사위 0회 회귀. (c) 근거 필드 + 게이트 예외를 '검증 가능한 신호'로 좁혀 방어.
- 전부 이월 → 재도전이 '기억으로 스킵'만 되어 재플레이 긴장 소실. 핵심만 이월 + 실행 긴장(제물 보호
  등)은 매번 재수행으로 균형.
- 1·2번은 '구조 신박성'과 별개 축 — 같은 GPT 라운드에 묶되 구현은 분리.
