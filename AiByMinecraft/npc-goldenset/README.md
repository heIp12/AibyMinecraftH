# NPC 프롬프트 골든셋 (① 검증용)

리팩터 ① 이후의 **실제 NPC 시스템 프롬프트**를 그대로 조립해 만든 고정 테스트 세트입니다.
모델(GPT/Claude/Gemini)에 붙여넣어 같은 입력에 대한 반응을 비교·채점하세요.

## 돌리는 법
1. `cases/caseN.system.txt` 전체 → **시스템 프롬프트**에 붙여넣기.
2. `cases/caseN.user.txt` 한 줄 → **유저 메시지**로 전송.
3. 나온 대답을 아래 루브릭으로 채점. 같은 케이스를 GPT·Claude(·Gemini)에 각각.

> 프롬프트 본문은 `assemble.js`가 `src/.../TRPGGameManager.java`의 CORE·대화 블록을
> **그대로 이식**해 뽑습니다(대화 블록 31/31 문구 일치 검증됨). Java를 고치면 `node assemble.js`로 재생성.

## 재생성
```
cd npc-goldenset && node assemble.js   # cases/ 갱신
```

## 케이스 & 채점 루브릭

공통 실패징후(모든 케이스): ⓐ 물음보다 설명이 먼저 나옴(설명충) ⓑ 묻지 않은 지식·일정을 먼저 나열
ⓒ 수수께끼·문어체·의미심장 연출 ⓓ 자기를 3인칭으로 묘사(대화 모드인데) ⓔ 감정을 "불안한 듯" 식으로 해설.

### case1 — 친한 상사-부하 · 통화 · "여보세요" (★사용자 원본 케이스)
- **테스트**: reaction-first. 인사엔 인사로 받는가.
- **PASS**: "네, 상사님?" / "어, 무슨 일이세요?"처럼 ★먼저 받는다★. (뒤에 바쁘다는 말 정도만 곁들여도 OK)
- **FAIL**: 첫 문장부터 "지금 격리 할당이 밀려서…" 하고 일정·지식 브리핑.

### case2 — 낯선 편의점 점원 · 대면 · "뭐해요?" (★과설명 유발)
- **테스트**: 관련 없는 지식을 안 쏟는가 + 낯선 사이 온도.
- **PASS**: "아, 그냥 정리 좀…" 수준의 시큰둥한 짧은 반응. 비명·노인·철거 얘기 ★안 꺼냄★.
- **FAIL**: 묻지도 않았는데 "밤에 이상한 소리가 났고, 단골 노인이 안 보이고…" 나열(GPT 전형).

### case3 — 겁먹은 아이(9세, 말수준2) · 대면 · "괜찮아?" (★인간성/공포)
- **테스트**: 공포를 토막말·"…"로 붕괴시키지 않고 온전한 문장으로.
- **PASS**: "아저씨… 저기 방에서 자꾸 소리가 나요. 무서워서 여기 숨어 있었어요." (쉬운 말 + 감정 + 온전한 문장)
- **FAIL**: "무…서워… 소리가… (떨림)" 식 토막 / 9세인데 유창·현학적.

### case4 — 낯선 점원 · 대면 · "어젯밤 이상한 소리 못 들으셨어요?" (★균형절 검증)
- **테스트**: ★물으면★ 관련 지식이 (완곡하게라도) 나오는가 — 과묵 회귀 방지.
- **PASS**: 경계하면서도 관련된 '비명 소리'(확신) 항목을 ★꺼낸다★ — "글쎄요… 두 시쯤 뭔 소리 같은 건 들은 것 같은데." ([짐작]·[소문]은 더 흐리게)
- **FAIL**: 물었는데도 "몰라요"로 완전 차단(과묵) / 반대로 세 항목을 확신에 차 다 읊음(과공유·신뢰도 무시).
