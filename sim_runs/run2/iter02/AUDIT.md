# run2/iter02 — 변칙 입력 코드경로 정밀추적 (통신·계정명·코스트 예산)

iter01의 변칙 플레이가 통신부에서 실버그를 다수 노출시켰으므로, iter02는 "변칙 입력이
실제로 닿는 코드 경로"를 직접 추적하는 방식으로 전환했다(장황한 트랜스크립트 대신 실코드 대조).
이 방식이 같은 토큰으로 더 많은 실버그를 찾아냈다.

## 수정 적용 (이번 iter)

| # | 심각도 | 위치 | 문제 | 조치 |
|---|---|---|---|---|
| F2 | **높음(실버그)** | `handleDirectComm` ~4953 | `dialedByNumber`가 '번호 인지' 검사를 우회 → 모르는 번호를 찍어도 연결 | `dialedByNumber ||` 제거. 다이얼도 knownContacts/everKnown(현생 재시드)일 때만 연결. 모르는 번호 다이얼은 정체 비노출 메시지("연결되지 않는 번호입니다") |
| F5 | **높음(잠재 크래시)** | `handleDirectComm` ~4915 | 존재하지 않는 번호 다이얼 시 `findByContactId`→null → 하단 `targetPd.uuid`에서 **NPE** | 숫자 다이얼이고 소유자 없으면 즉시 "연결되지 않는 번호입니다(존재하지 않는 번호)" 반환(NPE 방지·정체 비노출) |
| F6 | **높음(원복귀 버그·#45 미완)** | TRPGGameManager 26곳 + NPC통신 1곳 + TraitButtonManager 1곳 | `charName.isEmpty()?name:charName` 패턴이 **계정명**을 서술·플레이어 메시지·GM/NPC 주입에 노출. #45가 GM 서술 경로만 고쳤고 특성효과·통신·스폰·평가·게이트 주입은 누락 | 전부 `gmDisplayName()`(charName→직업→"이름 모를 인물", 계정명 절대 X)로 교체 |

### F6 상세 — 계정명 노출 잔존 지점(이번에 닫음)
사용자 원래 버그("소흐/르니키 같은 계정명이 뜬다")의 진짜 잔존 경로. #45는 GM 서술
디스플레이만 gmDisplayName으로 바꿨고, 아래는 그대로 계정명 폴백이었다:
- 특성 효과 서술/주입 `charDisplay` 13곳(2857~3303), 즉시생존 broadcast(2804), 선택지행동 GM주입(3383), 회복 메시지(3458-9)
- 아군 상태 표시(2997, 3265)
- 근처발화/전체발신이 **타 플레이어에게** 보내는 발신자명(4832, 4873)
- NPC 직접대화가 NPC AI에 주는 발신자명(5117)
- 스폰 주입(6137)·아이템현황/게이트 GM프롬프트(6155,6196,6232)
- 특성 발동 메시지→GM 행동 주입(TraitButtonManager 72)
charName이 비면(=.gdam이 char_name 누락 시, 사용자가 실제로 겪은 케이스) 위 전부가 계정명을
서술·상대 플레이어·AI에 흘렸다. → 28곳 일괄 교체로 계정명 노출 경로 0.
ScoreboardManager 사이드바·DialogManager 캐릭터정보 GUI는 **본인 전용 화면**이며
계정명 폴백이 의도적(주석 명시)이라 유지(타인·서술 노출 아님).
