# run2/iter03 — 서술 색상/줄바꿈 포매터 정밀추적 (NarrativeDelivery.format)

모든 서술이 통과하는 고-영향 함수 `NarrativeDelivery.format()`를 추적하다 **굽은 따옴표
오염(quote corruption)**으로 4개 변환이 전부 깨져 있던 것을 발견.

## F7 (높음·서식 회귀) — format() 정규식에 굽은 따옴표가 섞여 4개 변환 무력화

git 추적 결과: 원본 `3c718c0`(가독성 색상 도입)은 정상이었으나, `68dfac7`(스토리/엔딩
서식 개선)에서 해당 정규식 문자열 ★내부★가 굽은 따옴표(“ ”)로 자동 변환되며 오염.
Java 문자열 바깥 구분자는 곧은 따옴표라 컴파일은 됐지만(런타임 무력화), 다음이 전부 깨짐:

| 라인 | 의도 | 오염 상태 | 결과 |
|---|---|---|---|
| 138 | 굽은→곧은 따옴표 정규화 | `replace('”','”')` (자기 자신, **no-op**) | 굽은 따옴표 대사가 152의 청록 색칠을 못 받음 |
| 139 | 마크다운 헤더(#) 제거 | `“(?m)^…”` (리터럴 “ 뒤 ^ → **영원히 불일치**) | GM이 `#`/`##`를 내면 원문 그대로 노출 |
| 141 | 독백 `<-..->` → 회색 | `“<-…->”` (굽은 따옴표로 감싼 것만 매칭) | 독백 색칠 안 됨(GM은 굽은 따옴표로 안 감쌈) |
| 143 | 연출 `<...>` → 노랑 | `“<(?!-)…>”` 동일 | 연출/시스템 효과 색칠 안 됨 |

GM 프롬프트(TRPGGameManager 576·593·597행)는 실제로 `<효과>`·`<-내용->`·마크다운 금지를
지시 → 위 오염으로 **인게임에서 연출/독백 무색·`#` 헤더 노출**. #38/#31 서식 회귀.

### 조치
`3c718c0`의 정상 정규식 + `68dfac7`이 의도한 '독백 회색 분리'를 합쳐 복원:
```java
s = s.replace('“', '"').replace('”', '"');     // 정규화
s = s.replaceAll("(?m)^\\s*#{1,6}\\s*", "");   // 헤더 제거
s = s.replaceAll("<-([^<>\n]+)->", "§7<-$1->" + BASE_COLOR);   // 독백 회색
s = s.replaceAll("<(?!-)([^<>\n]+)>", "§e<$1>" + BASE_COLOR);  // 연출 노랑
```
`format()`는 프롤로그·턴 서술·에필로그(TRPGGameManager 4070)·다이얼로그(DialogManager
1213) 전부가 호출 → 전 구간 서식 복구. 전 파일 굽은따옴표-정규식 오염 재스캔 결과 0건.

## 결론
고-영향 포매터의 숨은 서식 회귀 1건(F7) 발견·복원. brace=0. 변칙 플레이가 아니라
"모든 서술이 지나는 길"을 따라간 코드 추적이 정적으론 안 보이던 런타임 무력화를 잡아냄.
