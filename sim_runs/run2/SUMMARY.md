# run2 종합 — 변칙 플레이 + 코드경로 추적 테스트 캠페인

저번 10h 세션처럼 "플레이어식 변칙 진행"으로 돌리되, 장황한 트랜스크립트 대신
**변칙 입력이 실제로 닿는 코드 경로를 직접 추적**하는 방식으로 전환했다.
같은 토큰으로 훨씬 많은 실버그를 잡았다.

## 발견·수정한 실버그/회귀 (5건, 전부 커밋·푸시)

| ID | 심각도 | 영역 | 내용 | 커밋 |
|---|---|---|---|---|
| F2 | 높음(실버그) | 통신 | @번호 직접 다이얼이 '번호 인지' 검사 우회 → 모르는 번호도 연결 | 2a34083 |
| F5 | 높음(잠재 크래시) | 통신 | 존재하지 않는 번호 다이얼 시 NPE | 2a34083 |
| F6 | 높음(원래 버그 잔존) | 계정명 | #45가 GM 서술만 고쳐, 특성효과·통신·NPC AI·스폰·게이트·평가 **30곳**에서 계정명 노출 잔존 → 전부 gmDisplayName() | 2a34083 |
| F7 | 높음(서식 회귀) | 서술 포매터 | format() 정규식 굽은따옴표 오염 → 연출/독백 무색+`#`헤더 노출(런타임 무력화) | b869e52 |
| **F8** | **높음(원래 버그 뿌리)** | 평가/압축 | **최종 평가 화면·평가AI·압축AI가 계정명 노출** — 사용자가 신고한 '에필로그 소흐/르니키'의 진짜 뿌리 | dc84151 |

**F6 + F8로 계정명은 이제 서술·통신·특성·스폰·게이트·평가·에필로그·컨텍스트압축 그
어느 AI/화면 경로로도 나가지 않는다(지시 의존이 아니라 ★구조적 차단★).** char_name이
.gdam에서 누락돼도 gmDisplayName(charName→직업→"이름 모를 인물")로 폴백.

## 코드 대조로 정상 확인 (수정 불필요, ~18개 계통)

통신 라우팅·다이얼·근처발화·전체발신 / 탭 자동완성(빈버퍼→전체, @접두사→필터) /
입력 디바운스(2.5s exact, 다이얼로그 입력은 예외) / Shift 빨리감기·서술중 입력차단 /
서술 청크 분할(문장경계·20자·2줄) / 능력 코스트 예산(쿨다운→파라미터→효과제거→스텟,
무한루프·NPE 안전) / 특성 등급클램프·선택 범위검사(0·음수·초과 안전) / 조우판정(같은
zone 시작 상호인지) / [합류] 등장 주입 / 아이템·열쇠·게이트(필드 가드·모호매칭 null) /
정체차용·직접통신(commDisplayName) / NPC 자율턴(zone필터·엿보기 동일zone) / 통신유인
(comms_dangerous) / 괴담 도청 주입 / 코루션 프로파일 학습(키=내부, 내용=발화) /
GdamGenerator char_name 강제 지시 / AI 품질 중품질(Sonnet) 기본.

## 보류(F1) — 변동 없음
co-location/zone 수렴·합류 런타임 신뢰성은 GM의 `<ZONE_UPDATE>` 발행에 의존(사용자
보류 항목). 시작 시점 zone은 seed대로 확정돼 조우판정은 정상. 해제 시: GM 프롬프트에
"이동 서술엔 ZONE_UPDATE 필수" 강제 + area 기준 협동 완화 권장.

## 미추적(잔여 저위험)
