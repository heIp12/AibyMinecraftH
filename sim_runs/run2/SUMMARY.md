# run2 종합 — 변칙 플레이 + 코드경로 추적 테스트 캠페인

저번 10h 세션처럼, 이번엔 "플레이어식 변칙 진행"으로 돌리되 장황한 트랜스크립트
대신 **변칙 입력이 실제로 닿는 코드 경로를 직접 추적**하는 방식으로 전환했다.
같은 토큰으로 더 많은 실버그를 잡았다.

## 발견·수정한 실버그/회귀 (4건, 전부 커밋·푸시)

| ID | 심각도 | 영역 | 내용 | 상태 |
|---|---|---|---|---|
| F2 | 높음(실버그) | 통신 | @번호 직접 다이얼이 '번호 인지' 검사를 우회 → 모르는 번호도 연결 | ✅ 수정 (커밋 2a34083) |
| F5 | 높음(잠재 크래시) | 통신 | 존재하지 않는 번호 다이얼 시 NPE | ✅ 수정 (2a34083) |
| F6 | 높음(원래 버그 잔존) | 계정명 노출 | #45가 GM 서술만 고쳐, 특성효과·통신·NPC AI·스폰·게이트·평가 **28+2곳**에서 charName 빈 값 시 계정명 노출 잔존 | ✅ 전부 gmDisplayName()로 차단 (2a34083) |
| F7 | 높음(서식 회귀) | 서술 포매터 | format() 정규식이 굽은따옴표 오염으로 연출·독백 무색+`#`헤더 노출(런타임 무력화, 컴파일은 정상) | ✅ 복원 (커밋 b869e52) |

F6는 사용자가 원래 신고한 "소흐/르니키 계정명이 뜬다"의 진짜 잔존 경로였다.
F7은 정적으론 안 보이는(컴파일 OK) 런타임 무력화로, "모든 서술이 지나는 길"을
따라가 발견.

| 통신유인(comms_dangerous) | ✅ @전화/@전체/무전 직후 괴담 반응 주입 |
| AI 품질 중품질 기본 | ✅ Quality.MEDIUM(Sonnet) 기본 |

## 보류(F1) — 변동 없음
co-location/zone 수렴·합류 런타임 신뢰성은 GM의 `<ZONE_UPDATE>` 발행에 의존.
사용자 보류 항목. 해제 시 GM 프롬프트에 "이동 서술엔 ZONE_UPDATE 필수" 강제 권장.

## 미추적(추가 캠페인 시 후보)
아이템/열쇠/게이트 통과, NPC 자율행동 턴, 점수·기여도 평가, GdamGenerator char_name
실측, 정체차용(impersonation) 응답, 오염 단계 상승 연출.
